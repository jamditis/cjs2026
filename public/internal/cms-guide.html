1. How it works

üîÑ
The content flow

The CJS 2026 website uses Airtable as a "headless CMS" - meaning you edit content in Airtable, and the website pulls that content automatically when it builds.

  1. You edit content in Airtable (text, dates, colors, etc.)
  2. Click "Update website" button in any row to trigger a deploy
  3. GitHub Actions automatically pulls the new content from Airtable
  4. The website rebuilds and deploys to Firebase (~60 seconds)
  5. Live site updates at cjs2026.web.app

The homepage hero section - controlled by Airtable's "hero" section content

No coding required! You can update most website text, dates, colors, and session information directly from Airtable without touching any code.

Some elements have a Color field. Available brand colors:

Never change the Field column! This is the unique ID that the code uses to find content. Changing it will break the website.

6. Publishing changes

The "Update website" button

Every row in Site Content has an "Update website" button. Clicking it triggers a GitHub Actions workflow that:

  1. Pulls all content from Airtable (both Site Content and Schedule tables)
  2. Generates static JavaScript files with the new content
  3. Builds the React application
  4. Deploys to Firebase Hosting
Changes go live in about 60 seconds. You can watch the progress in GitHub Actions if you have access.

When to click "Update website"

Scenario What to do
Changed one field in Site Content Click "Update website" in that row
Changed multiple fields Make all changes, then click the button once
Added/edited sessions in Schedule Go to Site Content and click any "Update website" button
Made a mistake Fix it, then click "Update website" again
Don't click repeatedly! Each click triggers a new deploy. Wait for the previous one to finish (~60 seconds) before clicking again.

7. Verifying your changes are live

After clicking "Update website", follow this checklist to confirm your changes are live:

Verification checklist

  • Wait at least 60 seconds - The deploy takes about a minute
  • Open the website - Go to cjs2026.web.app
  • Hard refresh the page:
    • Windows/Linux: Ctrl + Shift + R
    • Mac: Cmd + Shift + R
  • Check the specific section you edited
  • If it's not updated:
    • Try an incognito/private browser window
    • Try a different browser
    • Wait another 30 seconds and try again

If changes still don't appear

  1. Check that you clicked "Update website" after making your changes
  2. Verify the Visible checkbox is checked for the content
  3. Make sure you edited the Content field, not Name
  4. Contact Joe to check if the GitHub Action ran successfully

8. What you can (and can't) edit

‚úÖ You CAN edit in Airtable

  • Homepage headline, tagline, dates
  • Registration/ticket messaging
  • Timeline years, locations, themes
  • Stats (number of summits, cities, attendees)
  • "What to expect" cards
  • Footer text
  • All session information (titles, times, speakers, descriptions)
  • Colors for most elements
  • Show/hide content with the "Visible" checkbox

‚ùå You CANNOT edit in Airtable

  • Navigation menu items
  • Button text (like "Get tickets", "Sign in")
  • User dashboard content
  • Login/registration pages
  • Footer links
  • Page layouts and structure
  • Logos and images

These require code changes. Contact Joe or the developer.

Footer text is editable via Airtable; footer links require code changes

9. Tips & best practices

üìù Use sentence case for headlines

Write "Join us in Chapel Hill" not "Join Us In Chapel Hill". Only capitalize the first word and proper nouns.

üîç Preview before publishing

Double-check spelling and formatting in Airtable before clicking "Update website". Changes go live immediately.

üìÖ Use smart ordering

Use round numbers (10, 20, 30) for the order field so you can insert items later (15, 25) without renumbering everything.

üëÅÔ∏è Hide instead of delete

Instead of deleting content, uncheck "Visible" to hide it. You can bring it back later.

üîÑ Batch your edits

If you have multiple changes, make them all first, then click "Update website" once. This is faster and reduces deploy time.

‚è∞ Off-peak publishing

The site has a few seconds of downtime during deploys. Avoid publishing during peak hours if possible (9am, 12pm, 6pm).

10. Troubleshooting

‚ùì My changes aren't showing up

  1. Did you click "Update website"? Changes don't appear until you trigger a deploy.
  2. Wait 60-90 seconds for the deploy to complete.
  3. Hard refresh: Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac)
  4. Try a different browser or incognito mode to bypass cache
  5. Check that Visible is checked in Airtable

‚ùì The website looks broken

  1. Check if you accidentally changed a Field value (restore it from history)
  2. Make sure all required fields have values (Content, Section, Page)
  3. Contact Joe to investigate the GitHub Actions logs

‚ùì I need to undo a mistake

  1. In Airtable, click the expand icon on any cell
  2. Select "See record history" (or right-click the row)
  3. Find the previous value and restore it
  4. Click "Update website" to deploy the fix

‚ùì The "Update website" button doesn't work

  1. Make sure the button automation is enabled in Airtable (check Automations tab)
  2. Verify you have Editor access (not just Viewer)
  3. Contact Joe to verify the GitHub Actions workflow is configured

‚ùì Sessions aren't appearing on the schedule

  1. Check that visible is checked for the session
  2. Make sure day is set to "Monday" or "Tuesday" (exact spelling)
  3. Verify you clicked "Update website" in Site Content after adding sessions
  4. Hard refresh the /schedule page
